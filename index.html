About Me...

I'm Max Braun, a senior at Ball State University in Muncie, Indiana. I grew up in Evansville, Indiana and attended St. Benedict Cathedral where I was first introduced to the fascinating world of programming. I was given an assignment to make a game using Scratch and later a website using Weebly, and I was hooked. From there I graduated to Reitz Memorial High School where I continued my interest in development, and started doing occasional freelance web development work.

In August of 2018 I began my attendance at Ball State, majoring in Computer Science. I learned a lot about the software development industry that you don't learn from entry level freelancing or following online tutorials. I learned about unit testing, clean code, pair programming, time complexity, and much more. I'm very greatful for all I've been taught, the people I've met, and the experiences I've had at Ball State.

As my college career comes to an end and I enter the professional world, I hope to find a career in which I can continue to learn and grow as a developer. I, along with my girlfriend Macy, and our dogs, Blu and Harper, will soon be packing up and moving down to Florida. I'm very excited for the change and I hope to find my place as a developer in my new home.

What I've Made...

Pear Programming

I created Pear Programming as a place for entry level developers like me to share project ideas and work together to build their portfolios and teamwork skills. I built the client side using React, and the API using Node.js and Express, with MongoDB as the database. The client is hosted with Netlify and the API is hosted with Heroku. I also used Cloudinary to host the image uploads such as profile pictures and post images.

Passwords are hashed server-side and API routes are protected based on whether you are logged in or not. Please feel free to make an account and try it out!

This Website

I created this website in my spare time as an extension to my resume. Many of my old web development projects have been lost to the chaotic file management of my teenage self, so this website was a great way to wrap up some of what I learned back then into one website. I decided to forgo the use of a CMS like Wordpress for this site in order to not only have more precise control over the layout and functionality, but also to display my ability to build a responsive website from scratch.

WikiSearch

A project partner and I collaborated to develop and test a Java application which uses Wikipedia’s API to display page data. Given a term that exists as a page on Wikipedia, the application displays information about that page’s recent revisions and/or the pages top editors.
